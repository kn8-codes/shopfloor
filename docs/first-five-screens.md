# ShopFloor first five screens

## Goal
These are the first five screens worth building.
If these work, ShopFloor has a real heartbeat.

---

## 1. Neighborhood feed
### Purpose
Show open requests and recent field notes in one local stream.

### Must show
- request title
- category
- neighborhood
- urgency
- author handle
- status
- quick filter controls

### Why it matters
This is the front door.
It has to answer: "Is anything real happening here?"

---

## 2. Request detail
### Purpose
Give one practical problem enough room to be understood and answered.

### Must show
- title
- full description
- attached photos if any
- neighborhood
- category
- urgency
- budget/barter note
- responses thread
- button to respond
- button to mark resolved

### Why it matters
If this screen is weak, the whole product turns into vague posting sludge.

---

## 3. Shop card profile
### Purpose
Let people quickly judge whether someone is actually useful and local.

### Must show
- display name / handle
- neighborhood
- help style
- skills
- tools
- needs
- completed help count
- field note count

### Why it matters
Trust in ShopFloor comes from visible usefulness, not brand polish.

---

## 4. New request flow
### Purpose
Make it dead simple to ask for help without writing an essay.

### Fields
- title
- category
- neighborhood
- urgency
- description
- optional image
- optional budget note

### Why it matters
If asking for help feels like paperwork, people will bounce.

---

## 5. Field note creation / archive
### Purpose
Turn one solved problem into reusable local memory.

### Must show
- problem
- fix
- tools used
- cost
- time
- safety level
- neighborhood tip

### Why it matters
This is where the product compounds.
Without field notes, it is just another board.

---

## Immediate UI principle
All five screens should feel:
- mobile-first
- plain-language
- low friction
- practical, not polished-for-investors
