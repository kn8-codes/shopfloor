# Data retrieval triage

## What problem this helps with

Use this when the important thing is not the device itself, but the photos, documents, contacts, account access, messages, or evidence on it.

Data retrieval is different from repair. A repair attempt can make recovery harder. Treat the data as the priority until the owner says otherwise.

## Quick triage

1. Ask what data matters most.
2. Ask whether there is any backup: iCloud, Google, computer sync, SD card, external drive, or app-specific backup.
3. Do not browse personal files beyond what is needed for recovery.
4. If the device powers on, preserve battery and avoid unnecessary changes.
5. If storage may be failing, stop using the device and plan a careful copy.

## Tools and materials

- Owner consent and clear scope
- Known-good charger and cable
- Computer with enough storage
- External drive for copies
- [[Multimeter]] only if checking power accessories or simple charging issues

## Safety / stop conditions

Stop if the device has liquid damage, swelling battery, repeated boot loops, clicking or failing hard drive sounds, or data that may be legally sensitive.

Stop if you feel tempted to “just look around.” Privacy is not optional.

Do not run repair experiments on the only copy of important data.

## What to try first

Start with backups. Many recoveries are account recoveries wearing a hardware mask.

If backups fail, decide whether the device can be safely powered and copied. If not, document symptoms and recommend experienced help.

## When to ask for experienced help

Ask for experienced help when the storage device may be failing, the phone needs board-level repair, the data is high-value, or chain-of-custody matters.
