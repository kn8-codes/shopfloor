# ShopFloor Project Charter

## What this is

ShopFloor is a neighborhood repair and survival network for working people in Akron: local practical help, shop cards, tools/resources, requests, and field notes that turn fixes into reusable local knowledge.

## What this is not

- Not a place for unbounded agent invention.
- Not a parking lot for vague intentions.
- Not done until evidence exists.

## Current goal

Finish and verify the first real user loop: sign in, create shop card, post request, view real request page, then decide whether structured tools are in this sprint.

## Current phase

audit

## Stakeholders / users

- Nate: authority, priority, final judgment.

## Constraints

- Preserve working code and existing artifacts.
- Ask before destructive actions.
- Prefer small verified changes over heroic rewrites.

## Definition of done

- Acceptance criteria are met.
- Evidence is recorded in `docs/EVIDENCE.md`.
- `STATE.md` is updated.

## Stop conditions

Stop and ask Nate when:

- Scope changes materially.
- A destructive action is required.
- Credentials, payments, private data, or publishing are involved.
- The next step is ambiguous enough that guessing would create rework.

## Do not build / do not do

- Do not expand scope because a tool made it easy.
- Do not refactor unrelated working code.
- Do not claim completion without receipts.
